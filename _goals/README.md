# 目标详情页使用说明

## 📁 文件夹说明

`_goals/` 文件夹用于存放所有目标的详情页面。每个目标都有一个对应的 Markdown 文件。

## 📝 如何添加新目标详情页

### 1. 创建文件

在 `_goals/` 目录下创建一个新的 `.md` 文件，文件名使用小写英文和连字符，例如：
- `learn-rust.md`
- `fullstack-app.md`
- `tech-blogging.md`

### 2. 文件模板

```markdown
---
layout: post
title: 你的目标标题
permalink: /goals/your-goal-name/
date: 2026-02-11 10:00:00 +0800
categories: [分类1, 分类2]
tags: [标签1, 标签2, 标签3]
---

## 🎯 目标概述

简要描述你的目标...

## 📊 当前进度

- **状态**: 🚀 进行中 / ✅ 已完成 / 📋 计划中
- **完成度**: 60%
- **截止日期**: Q2 2026

## 📝 详细计划

### 阶段一
- [ ] 任务1
- [x] 任务2
- [ ] 任务3

## 💡 学习笔记

记录重要的知识点...

## 📚 参考资料

- [资料链接1](url)
- [资料链接2](url)

## ✅ 已完成成果

列出你的成果...

---

**下一步计划**: ...
```

### 3. 在 goals.yml 中添加链接

编辑 `_data/goals.yml`，为对应的目标添加 `url` 字段：

```yaml
study:
  - title: 你的目标标题
    desc: 目标描述
    tag: 标签
    progress: 50
    deadline: Q2 2026
    completed: false
    url: /goals/your-goal-filename  # 注意：不要加 .md 后缀
```

## 🎨 支持的 Markdown 功能

- **标题**: `##`, `###` 等
- **列表**: 有序列表和无序列表
- **任务列表**: `- [ ]` 和 `- [x]`
- **代码块**: 支持语法高亮
- **引用**: `>`
- **表格**: Markdown 表格语法
- **链接**: `[文字](url)`
- **图片**: `![alt](image-url)`
- **加粗**: `**文字**`
- **斜体**: `*文字*`
- **删除线**: `~~文字~~`

## 📋 分类建议

### categories（分类）
- 学习
- 项目
- 职业
- 个人成长

### tags（标签）
- 技术类: TypeScript, Rust, Docker, AI/ML
- 领域类: 前端开发, 系统编程, DevOps
- 类型类: 阅读, 实战, 认证, 健康

## 🔗 URL 规则

文件路径：`_goals/learn-rust.md`
访问URL：`/goals/learn-rust/`

注意：
- 文件名使用小写和连字符
- URL 会自动添加斜杠
- 在 goals.yml 中不要包含 `.md` 后缀

## 💡 最佳实践

1. **详细记录**: 包含计划、进度、笔记、成果
2. **定期更新**: 完成阶段性任务后及时更新
3. **添加代码示例**: 技术类目标可以添加代码片段
4. **链接资料**: 记录学习资源和参考链接
5. **反思总结**: 记录学习心得和遇到的挑战
